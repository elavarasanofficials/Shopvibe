# ShopVibe Workspace Rules

## Documentation Rule
- **CRITICAL:** Do NOT save plans, tasks, research notes, or design documents in the AI's internal `brain` folder.
- **ALWAYS** save, update, and manage all project planning and documentation files inside the `docs/` folder of this repository (`/run/media/lmx/LMX/Mix/Shopvibe/docs/`).
- This ensures the documentation is portable, version-controlled via Git, and accessible from any system.

## Workflow Rotation (The Documentation Cycle)
Follow this continuous cycle for any major feature or phase:
1. **Plan & Research:** Update files in `docs/` before writing code.
2. **Execute:** Build the feature.
3. **Reflect:** Update `docs/task.md` and `docs/implementation_plan.md` to match the exact implemented reality.
4. **Commit:** Commit the code AND the updated docs together.
